# 2026-06-22 AI新增楼盘功能

## 概述
在楼盘管理页面新增"AI新增楼盘"按钮，上传楼盘资料图片后通过 **腾讯云 OCR + 腾讯混元大模型** 自动识别并解析楼盘信息，自动填入表单。

## 技术方案

```
图片上传 → 腾讯云 OCR（文字提取）→ 腾讯混元大模型（语义解析）→ 自动填表
```

**为什么用 AI 解析而不是正则？**
- 楼盘资料格式多样（海报、文档截图、表格），正则只能匹配固定模式
- 混元大模型理解语义，能适配"项目名称"/"楼盘案名"/"项目案名"等多种表述
- 后续维护成本低，无需不断补充正则规则

## 后端改动

### 1. pom.xml - 新增依赖
```xml
tencentcloud-sdk-java-ocr:3.1.1379       <!-- OCR 通用印刷体识别 -->
tencentcloud-sdk-java-hunyuan:3.1.1379   <!-- 混元大模型 ChatCompletions -->
```

### 2. application.yml - 调大上传限制
```yaml
spring.servlet.multipart:
  max-file-size: 10MB     # 原 5MB
  max-request-size: 50MB  # 原 6MB （多图上传需要更大容量）
```

### 3. application-dev.yml - 新增 AI 配置
```yaml
ai:
  secret-id: ...          # 复用 COS 凭证
  secret-key: ...
  region: ap-guangzhou
  hunyuan-model: hunyuan-turbo
```

## 调试记录

### 问题1：500 服务器内部错误
- **原因**：multipart max-request-size 只有 6MB，多图上传超限
- **修复**：调整为 10MB/50MB

### 问题2：OCR 使用 ImageUrl 可能因 COS 桶无公开读权限而失败
- **原因**：上传 COS 后传 URL 给 OCR，但 COS 桶可能未开公开读
- **修复**：改为 base64 方式传给 OCR，先读文件字节后再上传 COS

### 问题3：错误信息不透明
- **原因**：GlobalExceptionHandler 兜底返回"服务器内部错误"
- **修复**：Controller 层 try-catch，返回具体错误原因

## 字段覆盖
AI 可识别全部 36 个楼盘字段，无法确定的字段设为 null 自动留空。

## 注意事项
- OCR + AI 调用耗时约 5-15 秒
- 混元大模型返回的 JSON 自动清洗 markdown 代码块标记
- 单张图片限制 10MB，与 multipart 配置一致
